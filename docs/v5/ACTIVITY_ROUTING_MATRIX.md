No V5 function creates a Meeting/Event. Demos are created by the rep (manual booking) or a booking integration; `handleMeetingEvent` (WF007) detects/processes the existing Event; `await_meeting` waits; `sendDemoReminder` (WF010c) only sends the reminder.



### email_required summary
- Email path: every cadence step (the email IS the step).
- Call path: only on `call:neutral`/`call:noanswer` (the accompanying follow-up).
- Manual path: only when a step is explicitly flagged `email_required` (default none).
- Event singletons (`:0:*`) send their template regardless of `Sequence_Type`.
